### 7. 导致AI不知道业务的根本原因

**根因排序：**

1. **FAISS索引缺失** — RAG检索永远返回空，AI无法获取任何知识库内容
2. **知识库内容极度匮乏** — 3个知识库中2个为空，唯一有数据的仅7个极短chunk
3. **System Prompt不含业务知识** — 仅说"你是PetSchool客服"，不包含课程列表/价格/FAQ等
4. **数据库查询范围过窄** — ContextService只查用户/宠物/已购课程，不查课程目录/FAQ/套餐
5. **无自动同步机制** — 课程变更后需手动触发同步，且同步后FAISS索引不更新
6. **意图识别不调用LLM** — 关键词未匹配时直接返回Other，不触发LLM分类

### 8. 修复优先级

| 优先级 | 问题 | 影响 |
|--------|------|------|
| **P0** | FAISS索引缺失，RAG检索不可用 | 所有RAG查询返回空 |
| **P0** | System Prompt不含业务知识 | AI无法回答任何业务问题 |
| **P0** | 课程目录未进入上下文 | "有哪些课程"无法回答 |
| **P1** | FAQ知识库为空 | 常见问题全部编造 |
| **P1** | 课程套餐/分类未同步 | 套餐/分类信息缺失 |
| **P1** | 无自动同步机制 | 数据变更后AI知识过期 |

## 最终建议：如果只能做3件事

### 第1件：重建FAISS索引 + 充实知识库

**为什么**：当前FAISS索引完全缺失，RAG检索形同虚设。即使有7个chunk也无法被检索到。这是最致命的P0问题。

**做什么**：
- 执行FAISS索引重建，为现有7个chunk生成向量
- 将课程完整信息（分类/套餐/大纲/适合犬种/价格）同步到知识库
- 创建FAQ文档（营业时间/地址/退款政策/购买流程等）
- 创建宠物百科文档（常见品种/饲养指南/疫苗知识等）

### 第2件：在System Prompt中注入实时业务数据

**为什么**：当前Prompt仅说"你是PetSchool客服"，不含任何业务知识。LLM只能用自身训练数据编造答案。

**做什么**：
- 在System Prompt中注入课程目录（名称/价格/分类/描述）
- 注入FAQ摘要（退款政策/购买流程/联系方式）
- 注入当前可用套餐和优惠信息
- 这相当于给AI一本"员工手册"，确保回答基于真实数据

### 第3件：扩展ContextService查询课程目录

**为什么**：当前CourseContext只查"已购课程"，不查"课程目录"。用户问"有哪些课程"时，AI无法从上下文获取课程列表，只能依赖RAG（但RAG也不可用）或LLM编造。

**做什么**：
- ContextService增加查询全部课程目录的方法
- 将课程目录注入System Prompt
- 增加OrderContext独立构建（退款/支付/订单详情）
- 增加FAQ快速匹配（不经过LLM，直接返回标准答案）

---

> **总结**：AI客服看似专业流畅，实则是"漂亮的空壳"——FAISS索引缺失导致RAG不可用，知识库内容极度匮乏，System Prompt不含业务知识，数据库查询范围过窄。AI的回答99%来自LLM自身编造，与PetSchool真实业务严重不符。修复核心是：**让AI能读到真实数据**。
